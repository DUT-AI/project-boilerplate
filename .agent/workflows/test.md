# /test

$ARGUMENTS

## Purpose

Run or design focused validation for frontend changes.

## Required Skills

Read `.agent/agents/test-engineer.md`, then read relevant skills:

Read in this order as needed:

1. `.agent/skills/test/SKILL.md`
2. `.agent/skills/code-review/SKILL.md` before completion claims

## Behavior

1. Identify what changed and which user-facing behavior is at risk.
2. Choose focused validation: unit, integration, UI, accessibility, build, or manual browser check.
3. Run available project commands when present.
4. Report commands, results, failures, and remaining risk.

## Examples

```text
/test validate this component
/test run final checks
/test check UI regressions
```
